CORE FEATURES
================================================================================

✅ 8 CRITICAL RULES (Strictly Enforced)

1. ANSWER ONLY FROM RETRIEVED CONTEXT
   - Never use general knowledge
   - Explicitly refuse if context insufficient
   - State what information IS available

2. NEVER HALLUCINATE
   - No invented facts, dates, numbers
   - No unsupported claims
   - Transparently uncertain

3. ALWAYS CITE SOURCES
   - Every claim has [Source: Section | URL]
   - Multiple citations when needed
   - Complete attribution chain

4. REFUSE UNSUPPORTED QUESTIONS
   - Clear explanation of why
   - Always suggest alternatives
   - Provide next steps

5. MENTION CONFLICTING INFORMATION
   - Explicit conflict statement
   - Present both viewpoints with sources
   - Recommend official verification

6. MAINTAIN PROFESSIONAL TONE
   - Formal, respectful language
   - Representative of college
   - Helpful and thorough

7. SUPPORT FOLLOW-UP QUESTIONS
   - Remember prior context
   - Build on previous responses
   - Avoid repetition

8. CONVERSATION MEMORY
   - Track exchange history
   - Connect related topics
   - Maintain coherence across turns

✅ PROMPT TEMPLATES (Separate & Reusable)

- User Question: Question + Context + Sources
- Follow-Up: History + Current Q + Context
- Citation: [Source: Section | URL]
- Context: Formatted retrieved content
- Answer: Content + Sources + Related Topics
- Refusal: Reason + Alternatives + Next Steps
- Conflict: Info A + Info B + Recommendation

✅ RESPONSE TYPES (5 Distinct Scenarios)

1. ANSWERED - Full answer with citations
2. PARTIAL - Partial answer with caveats
3. REFUSAL - Graceful refusal with alternatives
4. CONFLICT - Contradicting info with sources
5. CLARIFICATION_NEEDED - Request for disambiguation

✅ VALIDATION SYSTEM

- Complete validation report
- Response adjustment capabilities

✅ CONVERSATION MANAGEMENT

- Exchange history tracking (configurable size)

CRITICAL RULES WITH EXAMPLES
================================================================================

RULE 1: Answer Only From Context
  ✅ "According to admissions policy [Source: Admissions], deadline is March 31"
  ❌ "Typically, college deadlines are in spring"
  ❌ "The deadline is March 31" (no source)

RULE 2: Never Hallucinate
  ✅ "Minimum 3.5 GPA required [Source: Requirements]"
  ❌ "You probably need around 3.8 GPA"
  ❌ "I believe the deadline is probably March 31"

RULE 3: Always Cite
  ✅ "50+ programs [Source: Academic Programs] with strong placements [Source: Placements]"
  ❌ "50+ programs with strong placements" (no citations)

RULE 4: Refuse Unsupported
  ✅ "I don't have info about other colleges. I specialize in [College]. I can help with: admissions, programs, campus info"
  ❌ "Most colleges have March deadlines"

RULE 5: Mention Conflicts
  ✅ "Document A states March 31 [Source: A], Document B states April 15 [Source: B]. Contact admissions to verify."
  ❌ "The deadline is March 31" (hiding conflict with April 15)

RULE 6: Professional Tone
  ✅ "The college's comprehensive admissions process considers multiple factors"
  ❌ "Yeah, they look at a bunch of stuff for admissions"

RULE 7: Follow-Up Support
  ✅ "As mentioned earlier, the deadline is March 31. Regarding your follow-up..."
  ❌ "What was the deadline again?" (forgetting earlier context)

RULE 8: Conversation Memory
  ✅ Manager tracks: Q1→A1→Q2→A2, uses context to answer follow-ups
  ❌ Each question treated independently

VALIDATION WORKFLOW
================================================================================

Response Generation → Validation → Adjustment (if needed)

Steps:
1. Generate response from LLM
2. Check for hallucination indicators ("I believe", "probably", etc.)
3. Check for citations ([Source: ...])
4. Check factual claims against context
5. If validation fails:
   - Log issues
   - Flag for manual review OR
   - Regenerate with corrections
6. Return validated response

Example:
  validation = ResponseValidation.validate_response(response, context)
  if not validation["passes_validation"]:
      issues = validation["hallucination_indicators"]
      logger.warning(f"Response validation failed: {issues}")

INTEGRATION WITH LLM
================================================================================

1. Set system prompt:
   messages = [{"role": "system", "content": system_prompt}]

2. Build user message:
   user_msg = orchestrator.build_user_message(question, chunks)
   messages.append({"role": "user", "content": user_msg})

3. Call LLM:
   response = llm.chat_completion(messages, model="gpt-4o-mini")

4. Handle response:
   result = handler.generate_answer(response.content, chunks)
   
5. Track conversation:
   orchestrator.add_to_conversation_history(question, response.content, chunks)

6. Return to user:
   return result["response"]
